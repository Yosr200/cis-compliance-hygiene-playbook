# Remediation Tracking (Operational Playbook)

## Ticket fields (minimum)
- Asset / service name
- Control ID (CIS item)
- Finding description (plain language)
- Risk statement (business impact)
- Remediation guidance (what good looks like)
- Owner + due date (SLA)
- Status (Open / In progress / Blocked / Verified / Closed)

## Escalation rules (example)
- Tier 0/1 overdue → escalate to security management + platform lead
- Repeated overdue by same team → weekly steering meeting
